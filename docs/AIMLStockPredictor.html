🔍 1️⃣ Statistical Algorithms — “Market Thermometer”

🧮 mean()

  • Simple arithmetic average of prices or returns.
  • Used to smooth fluctuations and compare against current price (moving averages).

📉 stdev()

  • Measures volatility — how much the price bounces around its mean.
  • High stdev = risky, low stdev = stable.

🔁 pct() + logret()

  • pct() → simple percent return.
  • logret() → continuous compounding return, useful for volatility and regression.

👉 Together these form your descriptive analytics layer — they don’t predict, but describe current behavior accurately.

⚠️ 2️⃣ Risk Algorithm — “maxDrawdown()”

This is a non-linear algorithm that tracks the largest percentage drop from any peak to a subsequent trough in the last year.

Mathematically:

Drawdown(t)=Pt−max⁡(P0..t)max⁡(P0..t)\text{Drawdown}(t) = \frac{P_t - \max(P_{0..t})}{\max(P_{0..t})}Drawdown(t)=max(P0..t​)Pt​−max(P0..t​)​

  • Helps measure the worst-case loss scenario.
  • For conservative investors, this is the most important single metric.
  • It is not linear — it depends on the path of prices, not just averages.

📈 3️⃣ Trend Algorithm — Linear Regression (linReg)

The regression function fits a straight line through recent prices (usually last 60 trading days).

Mathematically:

y=a+b×xy = a + b \times xy=a+b×x

Where:

  • a = intercept
  • b = slope (average daily change)
  • r² = fit quality (how consistent the trend is)

Interpretation for TSLA:

  • Positive b → Uptrend (momentum building).
  • Negative b → Downtrend.
  • High r² → Strong, clean trend (not noisy).

This is your linear model, forming the base of future projections.

🧠 4️⃣ Signal Algorithm — Rule-Based Decision Logic

After calculating all metrics, the app applies a logical classifier to determine the stock’s “signal.”

Simplified Rules:

If (price > 50-DMA AND 50-DMA > 200-DMA) OR (1m > 0 AND 3m > 0) and drawdown > -50%

    → Signal = BUY

Else if (1m > -5%) → Signal = WATCH

Else → Signal = AVOID

  • Combines trend (linear regression) + momentum (returns) + risk (drawdown).
  • Produces a categorical output (BUY / WATCH / AVOID) — effectively a simple expert system.

📅 5️⃣ Projection Algorithm — Linear Extrapolation

Using the regression coefficients (a, b), the app forecasts future prices:

Pfuture=a+b×(current_index+N)P_{future} = a + b \times (current\_index + N)Pfuture​=a+b×(current_index+N)

for N = 30, 60, 90 days.

It’s not a deep learning model — it’s a deterministic linear projection assuming the recent slope continues.

✅ Example (TSLA):

  • Current slope (b) = +0.7 → roughly +$0.70/day
  • Current price ≈ $220
  • 90-day projection ≈ $220 + 0.7×90 = $283

📅 1. Why use 2 years instead of 1

When you only look at the last 12 months, you’re capturing a period that might not represent a “normal” market.
If that year was unusually aggressive — with sharp rallies, meme-stock surges, or rate shocks — it can distort your view of what’s typical for the stock.

By expanding to 2 years:

  • You smooth out those short-term anomalies.
  • You include both bullish and corrective phases.
  • Your metrics (trend, volatility, drawdown) become statistically more stable.

✅ For Tesla (TSLA), 2023–2024 was extremely volatile — massive rallies followed by sharp corrections.
A 2-year window gives a better sense of the true behavior of TSLA over different market conditions.

📈 2. Why this matters for conservative investors

Conservative investors care less about chasing big short-term moves — they focus on stability, predictability, and controlled risk.
A single aggressive year can make a stock look:

  • More profitable (due to short-term gains),
  • But also much riskier (due to large swings).

By using 2 years of data:

  • Volatility (stdev) is calculated over a broader base → it better reflects average risk.
  • Max drawdown includes previous dips → warns you of how deep losses can go.
  • Regression trend (linReg) gives a smoother, more realistic projection.

Essentially, it helps conservative investors separate temporary hype from sustained performance.

📈 Step 2: Calculate key numbers using the functions

🧮 1. mean()

👉 Average price over this period.

mean=(200+205+210+208+215+220)/6=209.67\text{mean} = (200 + 205 + 210 + 208 + 215 + 220) / 6 = 209.67mean=(200+205+210+208+215+220)/6=209.67

So the average TSLA price in this window is $209.67.

📉 2. stdev()

👉 Measures how much prices fluctuate around the average — this is volatility.

If the prices swing a lot (say between 150 and 300), standard deviation will be high → more volatile stock.
If they stay tight (like 205–210), stdev is low → stable stock.

For our sample, the volatility comes out around $7, which is modest.

🔄 3. pct(a, b)

👉 Calculates percentage change between two points.

Example: last price (220) vs price 21 trading days ago (say 200):

pct=(220/200)−1=0.10=10%\text{pct} = (220 / 200) - 1 = 0.10 = 10\%pct=(220/200)−1=0.10=10%

So TSLA gained 10% in the past month.

👉 Finds logarithmic returns for every day — small daily % moves, but calculated in a way that can be added over time.

📉 5. maxDrawdown()

👉 Finds the worst drop from the highest point.

Example:
TSLA peaked at $300, later dropped to $180.

Drawdown=(180/300)−1=−40%\text{Drawdown} = (180 / 300) - 1 = -40\%Drawdown=(180/300)−1=−40%

So Tesla’s max drawdown = -40% — meaning it lost 40% from its last peak before recovering.

📈 6. linReg()

👉 Fits a straight line through recent 60-day prices to see the trend and predict the future.

Let’s say the regression line has:

  • Slope (b) = +0.5
    → price rising $0.50 per day.
  • Intercept (a) = 190

Then:

  • 30-day projection = 190 + 0.5 × (current_index + 30) → ≈ $205
  • 60-day projection = ≈ $220
  • 90-day projection = ≈ $235

So the app predicts a gradual upward trend for TSLA.
